Getting started

PHP doesn’t need a main method, a class wrapper, or a compile step. Every .php file is an entry point. The interpreter runs top to bottom, and anything outside <?php tags is emitted as raw output.

PHP
<?php

echo "Hello, World!";
Java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

PHP ships a built-in server for local development. No Tomcat, no WAR files — just run this from your project root: 

php -S localhost:8000

Variables and types

Every variable starts with $. PHP is dynamically typed — no int, String, or boolean declarations. A variable holds whatever you assign to it, and the runtime coerces types in most expressions.

PHP
<?php

$name  = "Alice";
$age   = 30;
$price = 19.99;
$items = ["book", "pen"];

$result = $name . " is " . $age;
// Concatenation uses . not +
Java
String name  = "Alice";
int    age   = 30;
double price = 19.99;
String[] items = {"book", "pen"};

String result = name + " is " + age;
// + works for both addition and concat

PHP uses . for string concatenation, not +. The + operator always does numeric addition and will cast strings to numbers — a common source of bugs when switching from Java.

Control flow

if, else, while, for, and switch work almost identically. The big addition is foreach — PHP’s cleanest way to iterate arrays and maps.

PHP
<?php

$scores = [92, 78, 85];

foreach ($scores as $score) {
    echo $score . "\n";
}

$user = ["name" => "Alice", "role" => "admin"];
foreach ($user as $key => $value) {
    echo "$key: $value\n";
}
Java
int[] scores = {92, 78, 85};

for (int score : scores) {
    System.out.println(score);
}

Map<String, String> user = Map.of("name", "Alice", "role", "admin");
for (var entry : user.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}

Double-quoted strings interpolate variables directly: "Hello, $name" works without concatenation. Use curly braces for complex expressions: "{$user['name']}".

Arrays — also maps

PHP has one data structure that does everything: the array. An array can be an indexed list, an associative map (dictionary), or both at once. This is the biggest mental-model shift from Java, where List, Map, and Set are separate types with separate APIs.

PHP
<?php

$list   = ["red", "green", "blue"];
$map    = ["name" => "Alice", "age" => 30];
$nested = [
    ["id" => 1, "title" => "First"],
    ["id" => 2, "title" => "Second"],
];

// Common array functions
$count  = count($list);
$filtered = array_filter($list, fn($v) => $v !== "green");
$mapped   = array_map(fn($v) => strtoupper($v), $list);
Java
List<String> list = List.of("red", "green", "blue");
Map<String, Object> map = Map.of("name", "Alice", "age", 30);
List<Map<String, Object>> nested = List.of(
    Map.of("id", 1, "title", "First"),
    Map.of("id", 2, "title", "Second")
);

long count    = list.size();
var  filtered = list.stream().filter(v -> !v.equals("green")).toList();
var  mapped   = list.stream().map(String::toUpperCase).toList();

PHP ships with dozens of array functions (array_filter, array_map, array_reduce, array_merge) that serve the same role as Java streams. There’s no method chaining — you compose with nested function calls or assign intermediate variables.

String handling

PHP treats strings as byte sequences with a rich function library. There’s no StringBuilder — concatenation is cheap enough for most cases, and double-quoted interpolation covers the rest.

PHP
<?php

$length  = strlen("hello");
$upper   = strtoupper("hello");
$contain = str_contains("hello world", "world");
$replace = str_replace("cat", "dog", "the cat sat");
$parts   = explode(",", "a,b,c");
$joined  = implode("-", ["a", "b", "c"]);
$sub     = substr("hello", 1, 3);  // "ell"
Java
int    length  = "hello".length();
String upper   = "hello".toUpperCase();
boolean contain = "hello world".contains("world");
String replace = "the cat sat".replace("cat", "dog");
String[] parts = "a,b,c".split(",");
String joined  = String.join("-", "a", "b", "c");
String sub     = "hello".substring(1, 4);  // "ell"
Needle, then haystack. PHP string functions take the search argument first, the subject second: str_contains($haystack, $needle) is the modern form (PHP 8.0+), but older functions like strpos($haystack, $needle) follow the same order.

Classes and objects

PHP’s object model will feel familiar. Classes, constructors, inheritance, interfaces, and abstract classes all exist. The syntax differences are small: $this instead of just this, -> instead of . for member access, and visibility goes on each property individually.

PHP
<?php

class User
{
    public function __construct(
        private string $name,
        private string $email,
    ) {}

    public function greet(): string
    {
        return "Hi, {$this->name}";
    }
}

$user = new User("Alice", "alice@example.com");
echo $user->greet();
Java
public class User {
    private final String name;
    private final String email;

    public User(String name, String email) {
        this.name = name;
        this.email = email;
    }

    public String greet() {
        return "Hi, " + this.name;
    }
}

User user = new User("Alice", "alice@example.com");
System.out.println(user.greet());

PHP 8 introduced constructor property promotion — declare visibility, type, and name in the constructor signature and the property is created for you. No more repetitive $this->field = $field boilerplate.

Interfaces and inheritance work as you’d expect. PHP supports single inheritance with extends, multiple interfaces with implements, and traits for horizontal code reuse (like Java’s default methods on interfaces, but more flexible).

The web context

This is where PHP departs from Java most sharply. PHP was built for the web. Request data arrives in superglobals — associative arrays that are always in scope, no injection, no HttpServletRequest parameter binding.

PHP
<?php

// /profile.php?id=42
$id = $_GET["id"] ?? null;

// Form POST
$email = $_POST["email"] ?? "";

// Session
session_start();
$_SESSION["user_id"] = 42;
$userId = $_SESSION["user_id"] ?? null;

// Request method
if ($_SERVER["REQUEST_METHOD"] === "POST") {
    // handle form
}
Java (Servlet)
// /profile?id=42
String id = request.getParameter("id");

// Form POST
String email = request.getParameter("email");

// Session
HttpSession session = request.getSession();
session.setAttribute("user_id", 42);
Integer userId = (Integer) session.getAttribute("user_id");

// Request method
if ("POST".equals(request.getMethod())) {
    // handle form
}

Each HTTP request runs a fresh PHP process (or handles one in a pool). There’s no application state between requests unless you persist it — to a session, a database, or a cache. This shared-nothing architecture is the reason PHP scales horizontally with little ceremony, but it also means you can’t keep things in memory the way a long-running Java process can.

Errors and exceptions

PHP historically mixed errors and exceptions, but PHP 8 has largely unified them. Most internal errors now throw Error or Exception objects, and try/catch/finally works the same way it does in Java.

PHP
<?php

try {
    $result = 10 / 0;
} catch (DivisionByZeroError $e) {
    error_log($e->getMessage());
    echo "Cannot divide by zero.";
} finally {
    cleanup();
}
Java
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    logger.error(e.getMessage());
    System.out.println("Cannot divide by zero.");
} finally {
    cleanup();
}

PHP does not have checked exceptions. You are never forced to declare or catch an exception type. This matches modern Java practice (most teams avoid checked exceptions anyway), but it means you rely on documentation and static analysis to know what a method might throw.

Watch out for

These trip up Java developers more than anything else in their first week with PHP:

  1. == is loose comparison. "123" == 123 is true because PHP coerces the string to an integer. Use === (identical) for strict type-and-value comparison, just like you use .equals() vs == in Java.
  2. Arrays pass by value. Assigning an array to another variable creates a copy. Use &$array to pass by reference, or work with objects (which do pass by reference).
  3. No method overloading. You can’t define the same method twice with different signatures. Use default arguments, variadic parameters, or check types inside a single method body.
  4. null and ??. $value = $_GET["key"] ?? "default" is the null-coalescing operator. It checks if the left side is set and not null — much cleaner than nested isset() calls.
  5. Headers already sent. Once PHP emits any output (even a blank line before <?php), you can’t send HTTP headers or start a session. Keep your PHP files pure — no trailing whitespace outside tags.
